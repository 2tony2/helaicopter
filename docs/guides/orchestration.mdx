## The orchestration hub

Navigate to `/orchestration` for a unified view of repo-local workflow execution. The hub has two tabs:

### Conversation DAGs
Browse DAG summaries across all conversations — see which sessions spawned subagents, how deep the tree went, and aggregate token/cost stats.

### OATS runs
View Overnight OATS run records — each run shows:
- Run spec name and execution timestamp
- Task-level status (pending, running, succeeded, failed, timed out, skipped, blocked)
- DAG visualization of task dependencies
- Links to the underlying conversation threads

## Running an OATS spec

OATS run specs are markdown files that define multi-step agent workflows. Example:

```bash
# Preview the execution plan
uv run oats plan examples/my-spec.md

# Run locally
uv run oats run examples/my-spec.md
```

Each task in a run spec becomes a node in the execution DAG. Tasks can declare dependencies on other tasks, and OATS handles the scheduling.

## OATS activity indicators

The dashboard shows real-time status:
- **Active** — task is currently executing (heartbeat recent)
- **Stale** — task was active but heartbeat has aged out
- **Timed out** — task exceeded its time limit
- **Succeeded/Failed** — terminal states with result details

There is no separate orchestration control plane to start. The dashboard reads repo-local OATS runtime state and persisted run records directly.
