---
title: Helaicopter
description: A local-first platform for browsing, analyzing, and orchestrating Claude Code and OpenAI Codex conversations.
---

Helaicopter is a self-hosted analytics and operations platform that reads your AI coding conversations directly from `~/.claude/` and `~/.codex/`. Nothing leaves your machine.

## What you get

- **Analytics dashboard** — token usage, cost breakdowns, model mix, activity trends across all your Claude and Codex sessions
- **Conversation browser** — search, filter, and inspect every message, tool call, thinking block, and subagent thread
- **Conversation DAGs** — visualize parent/subagent relationships as directed acyclic graphs
- **Plan viewer** — browse saved implementation plans from `~/.claude/plans/`
- **Evaluation system** — run LLM-as-judge evaluations against conversation transcripts
- **Orchestration hub** — monitor OATS run specs and Prefect workflow executions
- **Database inspector** — view schema, table inventories, and refresh status for all backing stores

## Architecture at a glance

```
┌─────────────────────────────────┐
│   Next.js 16 Frontend (:3000)   │
│   React 19 · SWR · Recharts     │
│   Tailwind v4 · shadcn/ui       │
└────────────┬────────────────────┘
             │ HTTP (JSON)
┌────────────▼────────────────────┐
│   FastAPI Backend (:30000)      │
│   Pydantic v2 · SQLAlchemy 2    │
│   Hexagonal architecture        │
└────┬───────┬───────┬────────────┘
     │       │       │
  ~/.claude  SQLite  Prefect API
  ~/.codex   DuckDB  OATS runtime
```

The frontend never touches the filesystem directly — all data flows through the FastAPI backend, which owns the conversation parser, analytics aggregation, and database refresh pipeline.

## Quick start

```bash
# Start the backend
cd python && uv run uvicorn helaicopter_api.server.main:create_app --factory --port 30000

# Start the frontend (separate terminal)
npm run dev

# Refresh databases (loads conversations into SQLite + DuckDB)
npm run db:refresh
```
