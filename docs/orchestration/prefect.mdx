## How it works

The Helaicopter backend acts as a proxy to Prefect's REST API. This lets the frontend display Prefect data without needing direct access to the Prefect server, and allows the backend to enrich responses with OATS metadata and status normalization.

```
Frontend ──▶ Helaicopter Backend ──▶ Prefect API (127.0.0.1:4200)
                    │
                    ▼
              OATS metadata join
              Status tone mapping
```

## Prerequisites

You need a running Prefect server:

```bash
prefect server start
```

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `HELA_PREFECT_API_URL` | `http://127.0.0.1:4200/api` | Prefect API base URL |
| `HELA_PREFECT_API_TIMEOUT_SECONDS` | `10` | Request timeout in seconds |

## Endpoints

### Deployments

```
GET /orchestration/prefect/deployments
GET /orchestration/prefect/deployments/{deploymentId}
```

Returns registered workflow definitions. Each deployment includes:
- Name, description, schedule configuration
- Associated flow and tags
- Creation and update timestamps

### Flow runs

```
GET /orchestration/prefect/flow-runs
GET /orchestration/prefect/flow-runs/{flowRunId}
```

Execution history with:
- State (scheduled, pending, running, completed, failed, cancelled)
- Start time, duration, total run time
- Associated deployment and flow
- State transitions and logs

### Workers and work pools

```
GET /orchestration/prefect/workers
GET /orchestration/prefect/work-pools
```

Infrastructure monitoring:
- Worker heartbeat and last seen timestamp
- Work pool type, concurrency limits, and queue depth

## Data enrichment

The application layer (`application/prefect_orchestration.py`) adds:

- **Status tones** — maps Prefect states to normalized tones (`running`, `success`, `error`, `pending`, `unknown`) for consistent badge rendering
- **OATS correlation** — when a Prefect flow run originated from an OATS run spec, the response includes links to the OATS task DAG and underlying conversation threads

## Frontend hooks

```typescript
usePrefectDeployments()   // All deployments
usePrefectFlowRuns()      // Recent flow runs
usePrefectWorkers()       // Active workers
usePrefectWorkPools()     // Work pool configuration
```

These SWR hooks are defined in `src/hooks/use-conversations.ts` and poll via standard SWR revalidation.
