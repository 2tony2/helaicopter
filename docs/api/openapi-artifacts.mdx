## Artifact locations

The backend generates machine-readable API schemas in two formats:

| File | Format | Path |
|------|--------|------|
| JSON | OpenAPI 3.x | `public/openapi/helaicopter-api.json` |
| YAML | OpenAPI 3.x | `public/openapi/helaicopter-api.yaml` |

## Regenerating

After making changes to any router or schema in the backend:

```bash
npm run api:openapi
```

This runs the Python generator that introspects the live FastAPI app and writes updated artifacts to `public/openapi/`.

## Usage

These artifacts can be used for:

- **API documentation** — import into Mintlify, Swagger UI, or Redoc
- **Client generation** — feed into `openapi-generator` or `openapi-typescript` for typed client code
- **Testing** — validate API responses against the schema in CI
- **Playground** — use with tools like Bruno, Insomnia, or Postman for interactive exploration

## Schema conventions

- All response models are defined in `python/helaicopter_api/schema/`
- Field names use `snake_case` in the schema
- Enum values match Python string literals
- Optional fields are marked as nullable in the schema
