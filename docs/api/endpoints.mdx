## Health and operations

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/health` | Liveness probe — returns 200 if the server is running |
| `GET` | `/` | API gateway landing page |
| `GET` | `/gateway/direction` | Deployment info |

## Projects

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/projects` | List known projects with session counts |

Returns projects derived from conversation data. Each project includes the path and number of sessions.

## Conversations

| Method | Path | Query params | Description |
|--------|------|-------------|-------------|
| `GET` | `/conversations` | `project`, `days` | List conversation summaries |
| `GET` | `/conversations/{projectPath}/{sessionId}` | — | Full conversation detail |
| `GET` | `/conversations/{projectPath}/{sessionId}/dag` | — | Conversation DAG |
| `GET` | `/conversations/{projectPath}/{sessionId}/subagents/{agentId}` | — | Subagent transcript |
| `GET` | `/conversation-dags` | `project`, `days`, `provider` | DAG summaries with stats |

### Conversation summary fields
- `title` — auto-generated from first message
- `project_path` — encoded project directory
- `session_id` — UUID
- `provider` — `"claude"` or `"codex"`
- `model` — model identifier
- `input_tokens`, `output_tokens`, `cache_read_tokens`, `cache_write_tokens`
- `estimated_cost` — dollar cost based on model pricing
- `created_at`, `updated_at` — timestamps
- `has_subagents` — whether child threads exist

## Plans

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/plans` | List saved plan summaries |
| `GET` | `/plans/{slug}` | Plan detail with full markdown content |

Plans are loaded from `~/.claude/plans/`. Each plan includes title, creation date, and the full markdown body.

## Analytics

| Method | Path | Query params | Description |
|--------|------|-------------|-------------|
| `GET` | `/analytics` | `days`, `provider` | Aggregated token/cost/activity data |

Returns:
- `time_series` — hourly/daily data points with token counts and costs
- `model_breakdown` — token and cost distribution by model
- `cost_breakdown` — detailed cost attribution by provider and model
- `summary` — totals for the time window

## Databases

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/databases/status` | Artifact status, refresh state, table schemas |
| `POST` | `/databases/refresh` | Trigger full refresh pipeline |

The refresh endpoint re-parses all conversation files, updates both SQLite and DuckDB, invalidates backend caches, and regenerates schema documentation.

| Method | Path | Description |

## Orchestration — Prefect

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/orchestration/prefect/deployments` | All Prefect deployments |
| `GET` | `/orchestration/prefect/deployments/{deploymentId}` | Deployment detail |
| `GET` | `/orchestration/prefect/flow-runs` | Recent flow runs |
| `GET` | `/orchestration/prefect/flow-runs/{flowRunId}` | Flow run detail |
| `GET` | `/orchestration/prefect/workers` | Active workers with heartbeat |
| `GET` | `/orchestration/prefect/work-pools` | Work pool configuration |

These proxy the Prefect REST API (default `http://127.0.0.1:4200/api`), enriching responses with status tones and OATS metadata.
