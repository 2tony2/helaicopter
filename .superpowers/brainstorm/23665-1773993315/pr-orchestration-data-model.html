Design Section 2: Persisted Data Model

Extend existing Oats runtime and Prefect artifacts into a first-class branch / PR graph instead of creating a separate ledger.

Canonical storage
Run-scoped truth
.oats/runtime/<run_id>/state.json
Current execution snapshot, plus current feature-branch and task-PR state
Run-scoped audit trail
.oats/runtime/<run_id>/events.jsonl
Append-only lifecycle events for branch creation, PR creation, merge attempts, conflict resolution, and final review gating
Flow-run snapshots
.oats/prefect/flow-runs/<flow_run_id>/...
Execution-attempt snapshots that reference the same run_id and mirror relevant task branch / PR state for observability
Run-level fields
  • feature branch: long-lived run-owned branch, based on main
  • final PR: PR from feature branch to main, manual-final-gate status
  • stack status: building, awaiting-merge, resolving-conflict, ready-for-final-review, done
  • active operation: current PR create / merge / resolve step with session linkage
Task-level fields
  • parent branch reference: feature branch or upstream task branch
  • worktree path: stable reusable checkout for that task
  • branch snapshot: branch name, head sha, pushed / deleted status
  • task PR snapshot: PR number, URL, state, checks summary, merge outcome
  • operation history: create PR, merge attempt, conflict-resolution attempt, cleanup
Versioning and compatibility

Proposed

  • Introduce additive oats-plan-v2, oats-runtime-v2, and oats-run-v2 shapes.
  • Keep current integration_branch fields for compatibility, but treat them as the run’s feature branch in normalized backend responses.
  • Add explicit run_id linkage into Prefect local metadata and task checkpoints.

Avoid

  • Do not create an unrelated app-only git ledger before runtime artifacts are authoritative.
  • Do not make GitHub API responses the canonical stored model.
