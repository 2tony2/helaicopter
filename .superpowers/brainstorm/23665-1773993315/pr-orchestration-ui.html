Backend contract changes
  • Extend /orchestration/oats responses with featureBranch, finalPr, taskBranches, taskPrs, and operationHistory.
  • Add normalized summary fields for node badges: PR state, merge state, check rollup, conflict count.
  • Keep the existing DAG response, but add branch-parent and PR-parent relationships so the UI can render the stack without re-deriving it.
UI behavior
  • Run list cards show feature-branch health, task PR counts, and final-review status.
  • DAG nodes show compact git / PR badges directly on the node.
  • Detail pane adds tabs or sections for Execution, PR Stack, and Session Links.
  • Selecting a task highlights both its execution node and its stacked PR record.
