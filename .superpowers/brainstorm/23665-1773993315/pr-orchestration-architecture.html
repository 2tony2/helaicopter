Design Section 1: Control Plane

One Oats run owns one long-lived feature branch stack. Oats and Prefect persist the graph; Helaicopter renders and orchestrates it.

Ownership model
Run identity
Oats run = one feature branch, one feature PR, one resumable Prefect flow run family

Write path
  1. Planner builds task DAG and branch stack plan.
  2. Prefect task execution prepares or reuses the right worktree.
  3. Executor task writes code in its task branch.
  4. Oats creates the task PR and records PR metadata.
  5. Codex merge operator merges the PR into its parent branch.
  6. If merge fails, Oats launches a conflict-resolution step and retries.
  7. When stack completes, Oats opens the final feature PR to main.
Read path
  1. Oats runtime and Prefect artifact files are the canonical local record.
  2. Backend normalizes them into one orchestration response.
  3. Optional GitHub refresh enriches the stored snapshot, but does not replace it.
  4. UI shows both task execution state and branch / PR progression from the same graph.
Boundary decision

Oats + Prefect own

  • Run identity
  • Branch ancestry
  • Worktree lifecycle
  • PR creation + merge attempts
  • Conflict-resolution attempts

GitHub does not own

  • Primary orchestration truth
  • Resume semantics
  • Dependency graph meaning
  • Operator workflow history
